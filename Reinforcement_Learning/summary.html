Phase-2 · RL Battery Arbitrage
RL Battery Arbitrage Pipeline
Tariff schedule is read from CSV. The DQN agent observes state every hour, selects CHARGE / DISCHARGE / IDLE, and the environment steps forward. After 800 training episodes the agent converges to a buy-low discharge-high strategy.
How the model works
1
Load tariff schedule. A 24-row CSV maps each hour to a tariff (₹/kWh), status (OFF_PEAK / SHOULDER / PEAK), and severity score. This is the only external input — no live data feed.
2
Simulate the battery environment. BatteryEnv runs a 24-step episode (1 step = 1 hour). It tracks SoC, enforces physical limits (10–95%, max 50 kW), and applies ±0.3 tariff noise each step for robustness.
3
Agent observes state and picks an action. Each step the agent receives 4 normalised floats — time, battery level, tariff, severity — and outputs IDLE / CHARGE / DISCHARGE via epsilon-greedy selection.
4
Reward shapes the arbitrage strategy. Charging off-peak earns +0.5; charging during peak costs −3.0. Discharging during peak earns (kW × tariff)/10. This pushes the agent to buy cheap, sell dear.
5
DQN learns from experience replay. Each transition (s, a, r, s') is stored in a 20k buffer. Every step the agent samples 64 random transitions, computes TD targets, and backprops through a pure-numpy 4→128→128→3 network.
6
Target network stabilises training. A frozen copy of the Q-network updates every 50 steps, preventing the moving-target problem. Epsilon decays from 1.0 → 0.05 over 800 episodes.
7
Output: plots + terminal summary. After training, a greedy episode is run and saved as PNGs — tariff bars, SoC curve, action timeline. First run achieved a ₹1.03/kWh spread (charge avg ₹6.71, discharge avg ₹7.74).

enervia_rl.py
INPUT:  tariff_schedule   ← loaded from tariff_schedule.csv (24 rows: hour, tariff, status, severity)
OUTPUT: enervia_rl_training.png, enervia_rl_episode.png, terminal summary


── ENVIRONMENT SETUP ──────────────────────────────────────────────────

    BatteryEnv constants:
        CAPACITY_KWH  = 200.0
        MAX_POWER_KW  = 50.0
        EFFICIENCY    = 0.92
        SOC_MIN_PCT   = 0.10
        SOC_MAX_PCT   = 0.95

    state (4 floats):
        hour_norm     = hour / 23.0
        soc_norm      = soc / CAPACITY_KWH
        tariff_norm   = (tariff − 4.0) / 6.0
        severity_norm = severity / 10.0

    actions:  0=IDLE  |  1=CHARGE  |  2=DISCHARGE


── REWARD FUNCTION ────────────────────────────────────────────────────

    CHARGE:
        IF OFF_PEAK   →  reward = +0.5
        IF SHOULDER   →  reward = −0.5
        IF PEAK       →  reward = −3.0

    DISCHARGE:
        IF PEAK       →  reward = (disch_kw × tariff) / 10.0
        IF SHOULDER   →  reward = (disch_kw × tariff) / 20.0
        IF OFF_PEAK   →  reward = −1.0

    IDLE:
        IF OFF_PEAK and SoC < 50%   →  reward = −0.2
        ELSE                          →  reward = 0.0


── DQN AGENT ──────────────────────────────────────────────────────────

    Network: 3-layer numpy MLP
        input(4) → hidden(128) → hidden(128) → output(3)
        activation: ReLU,  init: Xavier

    Hyperparameters:
        gamma         = 0.95
        lr            = 5e-4
        batch_size    = 64
        eps_start     = 1.0
        eps_end       = 0.05
        eps_decay     = 0.995
        target_update = 50 steps
        replay_size   = 20,000


── TRAINING LOOP ──────────────────────────────────────────────────────

    FOR ep = 1 to 800:
        state  ←  env.reset()
        FOR hour = 0 to 23:
            action                    ←  agent.act(state)          // ε-greedy
            next_state, reward, done  ←  env.step(action)
            replay.push(state, action, reward, next_state, done)
            loss                      ←  agent.learn()             // sample batch, MSE update
            state                     ←  next_state
        IF ep % 50 == 0  →  run greedy eval episode


── OUTPUT ─────────────────────────────────────────────────────────────

    plot_training()  →  episode reward (raw + MA-30 + eval dots) + log loss curve
    plot_episode()   →  tariff bars (colour by status) + SoC line + action bars
    terminal:
        Total reward   : 289.33
        CHARGE  actions:  10  avg tariff ₹6.71
        DISCHARGE      :   9  avg tariff ₹7.74
        IDLE           :   5
        Tariff spread  : ₹1.03/kWh
M1
Tariff Schedule Loader
load_tariff_schedule(path)
Reads tariff_schedule.csv into a dict keyed by hour. Each entry holds tariff (₹/kWh), status string, and severity score. Single source of truth for all cost data.
24 rows hour → (tariff, status, severity) csv.DictReader
M2
Battery Environment
class BatteryEnv
Gymnasium-style 24-step episode simulating a 200 kWh BESS. Handles charge/discharge physics, SoC limits, and reward shaping per tariff period. Adds ±0.3 noise for robustness.
200 kWh · 50 kW η = 0.92 SoC 10–95% obs[4] floats
M3
DQN Agent
class DQNAgent + DQNNet
Pure-numpy DQN. Two hidden layers (128), ReLU, Xavier init. Epsilon-greedy exploration decaying 1.0→0.05. Online and target networks sync every 50 steps. Gradients clipped to ±1.
4→128→128→3 γ = 0.95 lr = 5e-4 batch 64 ε decay 0.995
M4
Training Loop
train(schedule, n_episodes=800)
Runs 800 episodes. Each episode: reset env, loop 24 steps — act, step, push to replay, learn. Greedy eval every 50 episodes. Returns agent, env, rewards, losses.
800 episodes eval every 50 replay 20k seed = 42
M5
Output & Plots
plot_training() · plot_episode()
Two PNG outputs on dark theme. plot_training: episode rewards (raw grey + MA-30 green + eval dots blue) and log-scale loss curve (purple). plot_episode: tariff bars coloured by period, SoC fill line with 10/95% limits, per-hour action bars. First run: total reward 289.33, tariff spread ₹1.03/kWh.
